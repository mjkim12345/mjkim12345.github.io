변수를 여러 개 따로 선언해서 학생 정보를 관리하다 보면 금방 한계가 온다. 학생이 10명이면 변수가 30개, 100명이면 300개가 필요하다. 자바의 클래스는 이 문제를 해결하기 위해 존재한다. 이 글에서는 클래스, 객체, 인스턴스의 개념을 처음부터 차근차근 정리해본다.

---

## 클래스란 무엇인가

클래스는 한마디로 **설계도**다. 붕어빵 틀을 생각해보면 이해가 빠르다. 붕어빵 틀 자체는 먹을 수 없다. 그 틀을 이용해서 찍어낸 팥 붕어빵이 실제로 존재하는 것이다. 클래스가 틀이고, 그 틀로 찍어낸 것이 객체다.

자바에서는 `int`, `String` 같은 기본 타입 외에도 내가 원하는 타입을 직접 만들 수 있다. 이걸 사용자 정의 타입이라고 하고, 그 설계도 역할을 하는 것이 바로 클래스다.

학생을 표현하는 클래스를 만들면 이렇다.

```java
package class1;

public class Student {
    String name;  // 이름
    int age;      // 나이
    int grade;    // 성적
}
```

`class` 키워드로 클래스를 선언하고, 안에 변수들을 정의한다. 이렇게 클래스 안에 선언된 변수를 **멤버 변수** 또는 **필드(Field)**라고 한다. 두 용어는 같은 의미로, 해당 클래스에 소속된 변수라는 뜻이다.

> **참고**  
> 멤버 변수와 필드는 완전히 같은 말이다. 자바 공식 문서에서는 주로 필드(Field)라는 표현을 쓰고, 실무에서는 둘 다 혼용한다.

## 객체 생성: new 키워드

클래스는 설계도일 뿐, 실제 메모리에 존재하는 것이 아니다. 실제로 데이터를 담으려면 클래스를 기반으로 **객체를 생성**해야 한다.

```java
Student student1 = new Student();
```

이 한 줄을 단계별로 쪼개보면 다음과 같다.

1. `new Student()` — Student 클래스 정보를 기반으로 메모리에 실제 객체를 만든다.
2. 객체가 생성되면 자바는 그 객체의 **참조값(주소)**을 반환한다. 예를 들어 `x001` 같은 메모리 주소다.
3. `Student student1` 변수에 그 참조값을 저장한다.

> **참고**  
> `new Student()` 코드 자체에는 이름이 없다. 생성된 객체에 나중에 접근하려면 그 주소를 어딘가에 저장해둬야 한다. 그 역할을 하는 것이 `student1` 변수다.

---

## 객체에 접근하기: 점(.) 연산자

생성된 객체의 멤버 변수에 값을 넣거나 읽으려면 **점(dot) 연산자**를 사용한다.

```java
student1.name = "학생1";
student1.age = 15;
student1.grade = 90;
```

`student1.name`이라고 쓰면, 자바는 `student1` 변수에 저장된 참조값(`x001`)을 읽고, 그 주소로 찾아가서 해당 객체의 `name`에 접근한다. 내부 동작을 풀어쓰면 이렇다.

```java
student1.name = "학생1"
// → x001.name = "학생1"  (참조값을 통해 실제 객체에 접근)
```

## 클래스, 객체, 인스턴스 — 용어 정리

세 용어가 헷갈리는 경우가 많다. 핵심만 짚어보자.

| 용어 | 의미 | 예시 |
|------|------|------|
| 클래스(Class) | 객체를 만들기 위한 설계도 | `Student` 클래스 |
| 객체(Object) | 클래스로부터 만들어진 실체, 서로 독립적인 상태를 가짐 | `student1`, `student2` |
| 인스턴스(Instance) | 객체와 같은 의미, 특정 클래스와의 관계를 강조할 때 사용 | "student1은 Student의 인스턴스다" |

객체와 인스턴스는 사실상 같은 말이다. "student1은 객체다"라고 하면 그냥 실체가 있다는 뜻이고, "student1은 Student 클래스의 인스턴스다"라고 하면 *어떤 클래스에서 나왔는지*를 명확히 하는 표현이다. 실무에서는 둘을 크게 구분하지 않고 혼용한다.

---

## 배열로 객체 여러 개 관리하기

객체도 타입이기 때문에 배열로 묶어서 관리할 수 있다.

```java
Student[] students = new Student[2];
students[0] = student1;
students[1] = student2;
```

배열에 저장되는 것은 객체 자체가 아니라 **참조값(주소)**이다. 자바에서 대입(`=`)은 항상 변수 안의 값을 복사해서 전달한다. `student1` 변수 안에는 참조값이 들어있으므로, 그 참조값이 복사되어 배열에 들어가는 것이다.

> **주의**  
> 변수에는 인스턴스 자체가 들어있지 않다. 인스턴스의 위치를 가리키는 참조값만 들어있다. 대입 시 인스턴스가 복사되는 게 아니라 **참조값만 복사**된다는 점을 반드시 기억하자.

---

## 핵심 요약

- **클래스** — 객체를 만들기 위한 설계도. 멤버 변수(필드)를 정의한다
- **객체 생성** — `new 클래스명()`으로 메모리에 실제 인스턴스를 만든다
- **참조값** — 변수에는 객체 자체가 아닌 메모리 주소(참조값)가 저장된다
- **점 연산자** — `변수.필드명`으로 객체의 멤버 변수에 접근한다
- **대입과 복사** — 대입 시 참조값만 복사된다. 객체가 복사되는 것이 아니다
